🔎 Attack Payloads to Try on Vulnerable Login

Username: ' OR '1'='1' --   Password: anything
Username: admin' --   Password: anything
Username: ' OR 1=1 --   Password: anything
→ Vulnerable Login: authentication bypassed, no valid credentials needed.
→ Secure Login: same payloads treated as literal strings, access denied.

✓ Valid test credentials:  admin / adminpass123  alice / alice2024  bob / bobsecret
What to observe during demo:
1) In vulnerable mode, user input becomes SQL code.
2) Payload ' OR '1'='1' -- can force WHERE clause to true.
3) In secure mode, same payload is bound as plain text data.
4) Result: no matching username, login blocked.
