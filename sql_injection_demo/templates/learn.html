SQL Injection Learning Guide

Educational explanation for this demo project (safe local testing only)

1) What is SQL Injection?

SQL injection happens when user input is concatenated into a SQL string, allowing the input to change query logic. In authentication, attackers try to force the WHERE clause to become true.

2) Vulnerable pattern

SELECT * FROM users WHERE username = '" + username + "' AND password_plain = '" + password + "'

Here, input is interpreted as SQL code.

3) Common demo payloads

  • ' OR '1'='1' --
  • admin'--
  • ' OR 1=1--

These payloads can bypass the vulnerable login route in this demo.

4) Why parameterized queries fix it

cursor.execute("SELECT * FROM users WHERE username = ?", (username,))

Input is bound as data, not parsed as SQL structure.

  • DB compiles query template first.
  • User value is inserted as a literal string.
  • Payload cannot alter SQL grammar, so bypass fails.

5) Extra hardening in this demo

  • bcrypt password hashing
  • Account lockout after repeated failures
  • Audit logging for security events
